# Miniguia de Estudos: Lógica de Programação e Estruturas de Dados com NotebookLM

## Contexto e objetivos
Este repositório contém o resultado do meu projeto prático desenvolvido para a **DIO**, utilizando o **NotebookLM** (IA do Google) como ferramenta de aprendizagem ativa. 

O objetivo central deste caderno temático é aprofundar o entendimento sobre **Lógica de Programação e Estruturas de Dados**, servindo como material de apoio para minha preparação para o programas de estágio/trainee e concursos na área de tecnologia.

**Objetivos específicos:**<br>
* Dominar conceitos de algoritmos e complexidade.
* Organizar uma base de conhecimento consultável para revisões rápidas.
* Praticar engenharia de prompts voltada para educação técnica.

## Miniguia de estudo (entrega final)

### 1. Resumo estruturado
A lógica de programação não é sobre decorar comandos, mas sobre a **decomposição de problemas**.
* **Entrada de Dados:** Como o sistema recebe a informação.
* **Processamento:** O uso de operadores lógicos (`AND`, `OR`, `NOT`) e estruturas condicionais (`IF/ELSE`, `SWITCH`) para transformar dados.
* **Saída:** O resultado final esperado entregue ao usuário ou sistema.

### 2. Glossário de conceitos chave
* **Algoritmo:** Uma sequência finita e bem definida de passos para resolver um problema.
* **Variável:** Um espaço alocado na memória para armazenar dados que podem ser alterados durante a execução.
* **Recursividade:** Técnica onde uma função chama a si mesma para resolver subproblemas menores do mesmo tipo.
* **Big O Notation:** Linguagem matemática usada para descrever a eficiência e performance de um algoritmo à medida que o volume de dados cresce.

### 3. Prompts reutilizáveis
Comandos para apoiar revisões futuras:
* **Explicação Simples:** "Explique o conceito de **[CONCEITO]** usando uma analogia do mundo real para facilitar o entendimento."
* **Autoavaliação:** "Crie 5 perguntas de múltipla escolha baseadas nestes textos para testar meu conhecimento sobre o assunto estudado."
* **Análise de Código:** "Analise o código abaixo e identifique possíveis erros de lógica segundo as boas práticas das fontes fornecidas."
